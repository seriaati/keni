# Product

## Register

product

## Users

Three overlapping groups: tech-savvy self-hosters who want full data control, casual personal-finance people who want low-effort logging, and expats or multi-currency users managing money across borders. All share one trait — they want to record transactions without friction and review them without overwhelm.

## Product Purpose

Keni is a personal finance tracker built for detailed transaction logs with minimal effort. AI handles the heavy lifting (parsing text, images, PDFs, voice) so users spend seconds logging, not minutes. Success looks like a user who actually sticks with it — not one who opens the app twice then abandons it.

## Brand Personality

Elegant, minimal, calm, professional. Quiet confidence — not sparse for the sake of it, but because every element earns its place. The interface recedes so the data can speak.

## Anti-references

- Loud finance dashboards (Mint, Personal Capital) with metric grids, gradient cards, and badge-heavy UI
- Neon or crypto-aesthetic wallet apps
- Over-animated productivity tools that celebrate every action
- Cluttered settings panels with toggles for toggles

## Design Principles

1. **Friction is the enemy.** Every interaction should feel like the shortest path to done. If a user has to think about the UI, it has failed.
2. **Data, not decoration.** Visuals exist to surface information — not to impress. Suppress chrome when it competes with content.
3. **Calm by default.** No urgent colors, no flashing states, no overwhelming density. The app should feel like a well-organized notebook, not a command center.
4. **Consistent, not rigid.** Follow the established visual language (cream/forest/amber, DM Sans + Instrument Serif) but apply it with judgment — don't force a pattern where it doesn't fit.
5. **Trust through precision.** Financial data demands accuracy in display: correct formatting, correct locale, no rounding surprises. Small errors destroy trust fast.

## Accessibility & Inclusion

No specific WCAG level mandated. Maintain sufficient contrast (especially forest-on-cream), keyboard navigability for all interactions, and respect system reduced-motion preference for animations.
